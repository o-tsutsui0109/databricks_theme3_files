# 解約防止 次アクション提案プレイブック

## 対象CSV

`genie_saas_customer_usage.csv`

## 目的

SaaS顧客の利用状況、問い合わせ、満足度、更新期限をもとに、解約防止の次アクションを提案する。

## 主な判断指標

- active_users
- login_count
- feature_ai_usage
- api_calls
- tickets_opened
- avg_resolution_hours
- csat
- mrr
- renewal_due_in_days
- churn_risk_label
- churn_flag

## 優先度ルール

### Priority A: 即時対応

以下のいずれかに該当する場合。

- `churn_flag = 1`
- `churn_risk_label = High` かつ `renewal_due_in_days <= 30`
- `csat < 3.0`
- `tickets_opened >= 10`
- `avg_resolution_hours >= 25`
- 更新期限30日以内で利用定着が弱い

### Priority B: 早期対応

以下のいずれかに該当する場合。

- `churn_risk_label = Medium`
- `csat < 3.6`
- `tickets_opened >= 7`
- `avg_resolution_hours >= 15`
- `renewal_due_in_days <= 60`
- AI機能利用が少ない

### Priority C: 監視継続

以下に該当する場合。

- `churn_risk_label = Low`
- CSATが4.0以上
- チケットが少ない
- 利用量が安定している

## 顧客別アクション例

### 首都建設

状況:

- Standardプラン
- アクティブユーザー40
- チケット10件
- 平均解決28時間
- CSAT 2.8
- 更新12日
- 解約リスクHigh
- 解約フラグ1

優先度: Priority A

推奨アクション:

1. AMとCS責任者で解約理由を確認する
2. 未解決チケットを分類し、業務影響が大きい順に対応する
3. 24時間以内に顧客へ改善方針を連絡する
4. 再提案可能性がある場合は、短期改善プランを提示する
5. 今後の類似顧客向けに解約理由をナレッジ化する

## 推奨アクション分類

### 利用定着が弱い場合

- 管理者向けオンボーディングを再実施
- 部署別ユースケースを提示
- AI機能の短時間トライアルシナリオを案内
- 利用状況レポートを顧客と共有

### 問い合わせが多い場合

- チケット分類
- FAQ化
- 操作トレーニング
- サポート優先キュー
- 原因別改善計画

### CSATが低い場合

- 顧客責任者への謝意と改善方針提示
- 未解決課題の棚卸し
- 次回連絡期限の明示
- CS責任者の同席

### 更新期限が近い場合

- 更新前面談
- 短期改善プラン
- 利用価値の再整理
- 導入効果レポート作成
