# 納期遅延対策 次アクション提案プレイブック

## 対象CSV

`genie_logistics_delivery.csv`

## 目的

配送実績データから、納期遅延の原因を分類し、顧客連絡、代替対応、再発防止策を提案する。

## 主な判断指標

- customer_region
- warehouse_id
- warehouse_name
- carrier_name
- product_category
- promised_delivery_date
- actual_delivery_date
- delay_days
- shipping_cost
- inventory_shortage_flag
- urgent_flag
- weather
- delay_reason

## 優先度ルール

### Priority A: 即時対応

以下のいずれかに該当する場合。

- `delay_days >= 3`
- `urgent_flag = 1` かつ `delay_days >= 1`
- `inventory_shortage_flag = 1` かつ `delay_days >= 2`
- 顧客への約束納期を大きく超過している
- 高額配送または重要顧客向け配送

### Priority B: 早期対応

以下のいずれかに該当する場合。

- `delay_days = 2`
- 配送会社遅延
- 出荷集中
- 天候影響が予測可能だった
- 同一倉庫または同一配送会社で遅延が複数発生

### Priority C: 監視継続

以下に該当する場合。

- `delay_days = 0`
- 遅延理由が「なし」
- 緊急フラグなし

## 遅延理由別アクション

### 在庫不足

対象例:

- O1003: 九州、名古屋DC、食品、3日遅延、在庫不足
- O1008: 九州、名古屋DC、家電、4日遅延、在庫不足

推奨アクション:

1. 代替倉庫に在庫があるか確認する
2. 顧客へ納期変更と代替案を連絡する
3. 対象商品の安全在庫を見直す
4. 九州向け出荷で名古屋DCの在庫不足が続いていないか確認する
5. 需要予測または補充点の見直しを行う

### 配送会社遅延

対象例:

- O1002: 関西、東西物流、家電、2日遅延
- O1009: 中部、東西物流、食品、1日遅延

推奨アクション:

1. 配送会社へ遅延理由と復旧見込みを確認する
2. 代替配送会社への切り替え条件を確認する
3. 顧客へ最新到着予定を連絡する
4. 同一配送会社の遅延率を週次で確認する

### 天候

対象例:

- O1005: 北海道、北辰運輸、暖房、2日遅延、強風
- O1011: 北海道、北辰運輸、家電、3日遅延、大雨

推奨アクション:

1. 天候による配送制限を確認する
2. 顧客へ不可抗力要因と見込み日を伝える
3. 北海道向け配送では予備日を含めた納期設計を検討する
4. 高優先度商品は前倒し出荷を検討する

### 出荷集中

対象例:

- O1006: 関東、大阪DC、東西物流、アパレル、緊急フラグあり、2日遅延
- O1012: 関西、東京DC、東西物流、食品、2日遅延

推奨アクション:

1. 出荷波動が集中した時間帯を確認する
2. 緊急フラグありの注文を先に処理する
3. 倉庫作業人員または配送枠の一時増強を検討する
4. 出荷締め時間やピッキング順序を見直す

## 重要注文の例

### O1008

状況:

- 九州向け
- 名古屋DC
- 南海配送
- 家電
- 4日遅延
- 在庫不足

優先度: Priority A

次アクション:

1. 顧客へ遅延連絡と最新予定日を即時連絡
2. 代替在庫の確認
3. 代替配送または分納の検討
4. 在庫不足再発防止策の作成

### O1006

状況:

- 関東向け
- 大阪DC
- 東西物流
- アパレル
- 緊急フラグあり
- 2日遅延
- 出荷集中

優先度: Priority A

次アクション:

1. 緊急注文として優先出荷
2. 倉庫責任者に作業状況を確認
3. 顧客へ到着見込みを連絡
4. 今後の緊急注文処理ルールを見直す
