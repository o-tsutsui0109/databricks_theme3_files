# 品質改善 次アクション提案プレイブック

## 対象CSV

`genie_manufacturing_quality.csv`

## 目的

製造ラインの不良率、停止時間、保全状況、設備、サプライヤー、温湿度をもとに、品質改善アクションを提案する。

## 主な判断指標

- plant_name
- line_id
- product_name
- shift
- planned_qty
- actual_qty
- defect_qty
- defect_rate
- downtime_minutes
- maintenance_done_flag
- equipment_id
- supplier_name
- temperature_c
- humidity_pct
- quality_alert_flag

## 優先度ルール

### Priority A: 即時対応

以下のいずれかに該当する場合。

- `quality_alert_flag = 1`
- `defect_rate >= 0.04`
- `downtime_minutes >= 100`
- 保全未実施かつ不良率が高い
- 同一設備または同一ラインで異常が再発している

### Priority B: 早期対応

以下のいずれかに該当する場合。

- `defect_rate >= 0.03`
- `downtime_minutes >= 80`
- 夜勤帯で品質が悪化
- 温度30度以上または湿度70%以上
- 特定サプライヤーで異常が複数発生

### Priority C: 監視継続

以下に該当する場合。

- `quality_alert_flag = 0`
- `defect_rate < 0.02`
- 停止時間が短い
- 保全実施済み

## データから見える主な傾向

### 夜勤帯の品質悪化

品質アラートありのデータには、夜勤が多く含まれる。  
夜勤帯では、停止時間が長く、不良率が高くなる傾向がある。

対象例:

- 千葉工場 L02 モーターB 夜勤
- 名古屋工場 L03 ギアB 夜勤
- 福岡工場 L02 バルブA 夜勤

### 保全未実施と品質悪化

品質アラートありの行では、保全未実施が多い。  
保全未実施、停止時間増加、不良率上昇が重なる場合は、設備点検を優先する。

### 温湿度の影響

温度30度以上、湿度70%以上の条件で品質アラートが発生している。  
高温多湿環境では、材料状態、設備安定性、作業環境の確認が必要。

### サプライヤー要因

北海金属が関係する行で品質アラートが複数見られる。  
ただし、原因をサプライヤーと断定せず、設備、シフト、保全、温湿度と合わせて確認する。

## 重要ケース

### 千葉工場 L02 モーターB 夜勤

状況:

- 実績870 / 計画1000
- 不良48
- 不良率5.52%
- 停止130分
- 保全未実施
- 設備EQ205
- サプライヤー北海金属
- 温度31度
- 湿度78%
- 品質アラートあり

優先度: Priority A

推奨アクション:

1. EQ205の緊急点検を実施
2. 夜勤帯の作業記録を確認
3. 北海金属ロットの受入検査結果を確認
4. 温湿度管理ログを確認
5. 次回生産前に保全を完了
6. 品質会議で再発防止策を決定

### 福岡工場 L02 バルブA 夜勤

状況:

- 実績790 / 計画900
- 不良35
- 不良率4.43%
- 停止110分
- 保全未実施
- 設備EQ212
- 温度30度
- 湿度75%
- 品質アラートあり

優先度: Priority A

推奨アクション:

1. EQ212の保全計画を前倒し
2. 夜勤責任者へのヒアリング
3. 高温多湿時の作業条件を確認
4. 良品・不良品のサンプル比較
5. 標準作業手順の逸脱有無を確認

## 原因仮説の書き方

原因が確定していない場合は、以下のように表現する。

- 「保全未実施が不良率上昇に影響した可能性があります」
- 「夜勤帯での作業条件または設備状態が関係している可能性があります」
- 「高温多湿環境が品質に影響した可能性があります」
- 「サプライヤー要因は候補の一つですが、設備・環境要因と合わせて確認が必要です」

## 現場向け指示テンプレート

> 対象ラインで不良率と停止時間の上昇が確認されています。まず設備点検、保全履歴、作業記録、材料ロット、温湿度ログを確認してください。原因が確定するまでは、同一条件での連続生産を避け、必要に応じて品質保証部門へ確認を依頼してください。
